# Foreshadowing Ledger

> **Purpose**: Track every plant and its payoff. AI continuity checks reference
> this file to ensure promises made to the reader are kept. A "plant" is any
> detail, image, or line that sets up a later revelation. A "payoff" is where
> that promise is fulfilled.
>
> **Statuses**: `planted` (waiting for payoff), `paid-off` (resolved),
> `implicit` (resolved through subtext, not explicit scene), `cut` (removed
> from manuscript).

## Instructions

1. **Add plants immediately** when you write them. Do not wait for the payoff.
2. **One row per thread**. If a thread has multiple plants or payoffs, list the
   primary ones and add detail in the description.
3. **Review before writing climactic chapters.** Check which threads are still
   `planted` and decide whether they need resolution.
4. **Never delete rows.** If a thread is cut from the manuscript, change its
   status to `cut` and note why in the description.
5. AI continuity tools scan this file. Keep the table format exact — do not
   add extra columns or change the header names.
